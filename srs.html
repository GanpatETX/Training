Applicant Tracking System (ATS)
Systems Requirement Document

Aerospace Company – Enterprise Recruitment

1. Executive Summary

This document defines the complete system requirements for an Applicant Tracking System (ATS) designed for an aerospace company. The solution automates the hiring requisition approval chain (Project Directors → Chief of Staff → Founders → PTC) and manages five recruitment stages (Screening, Fitment Evaluation, Technical Interview, PTC Round, Founder Round). Key features include role-based access (Admin, PTC, Hiring Manager, Directors, Founders), configuration of rejection remarks (minimum character limit), email template management, Zoho People (teams/leads) and Zoho Bookings (scheduling) integration, candidate uniqueness via email, multi-pipeline support per candidate, and two data views (Kanban / Tabular).

2. System Context & High-Level Architecture

Actors: Candidates (external), PTC/HR, Hiring Managers, Directors, Founders, Admin.
External integrations: Zoho People API (team/department/lead data), Zoho Bookings API (virtual interview scheduling), SMTP/Email service.
Data storage: Relational DB (candidates, pipelines, requisitions), object storage (CVs, task submissions).

3. Functional Requirements

3.1 Hiring Requisition Module

HR-02Approval workflow sequential: (1) any Project Director approves (notify all), (2) Chief of Staff, (3) any Founder (notify all), (4) PTC final approval.
HR-03Each stage captures: Status (Approved/Rejected) + Remarks (mandatory). PTC configures min character count per stage.
HR-04Rejection at any stage terminates workflow & notifies requisition creator.
HR-05After PTC approval, a Job Posting is automatically created for inbound applications.

3.3 Recruitment Pipeline (5 Stages)

StageOwnerKey Actions
1. ScreeningPTC / HRConfigurable fields (demographics, PTC Qs, technical Qs, CV upload PDF only)
2. Fitment EvaluationHiring ManagerAssign task (up to 5 saved tasks per role); candidate uploads PDF/ZIP
3. Technical InterviewHiring ManagerSchedule via Zoho Bookings; capture result + remarks
4. PTC RoundPTCVirtual interview; capture result + remarks
5. Founder RoundAny FounderVirtual interview; capture result + remarks

3.4 Pipeline Rules (All Stages)

  • PL-01 Outcome = Approved → move next stage; Rejected → exit pipeline.
  • PL-02 Rejection requires reason with configurable minimum character count (PTC setting).
  • PL-03 Candidate email notifications for shortlist/rejection start from Stage 2 (Fitment). Stage 1 (Screening) does NOT send candidate email.
  • PL-04 Referrals: start at Stage 2; PTC may skip Stage 2 (reasons: experienced, urgent).
  • PL-05 Hiring Manager is responsible for Fitment task & Technical interview.
  • PL-06 Internal stage communication emailed to: Hiring Manager, Supporting Tech Team Members, careers@ email group.
  • PL-07 PTC can dynamically select supporting team members per candidate via Zoho People API (teams/departments/leads).

3.5 Candidate Uniqueness & Multi-Pipeline Support

  • CU-01 Candidate email is unique identifier – system prevents duplicate profiles.
  • CU-02 Candidate can apply to same role once every N months (N configurable by PTC, default 3 or 6).
  • CU-03 Candidate may apply to multiple roles across departments → separate pipelines.
  • CU-04 Multiple applications within same department share a single Fitment Evaluation task. Pipeline diverges after that stage.

3.6 Authentication & Session Management

  • AUTH-01 Candidates: email-based code (magic link/OTP) – no username/password.
  • AUTH-02 Candidate session timeout: 15 minutes inactivity; TLS 1.3, HTTP-only cookies.
  • AUTH-03 Staff (Hiring Manager, PTC, Directors, Founders): register via email OTP, then set password.
  • AUTH-04 Staff login: email + password. Future SSO via Zoho integration.
  • AUTH-05 Admin account bootstrapped initially with highest privileges.

3.8 Screening Stage – Configurable Fields

  • SCR-01 PTC configures Screening form per role: demographics, 2 PTC custom questions, 3 technical custom questions, CV upload (PDF only).
  • SCR-02 Versioning: PTC saves multiple versions, views older versions, sets a default active version.

3.9 Fitment Evaluation Stage

  • FE-01 Hiring Manager can create/upload up to 5 tasks per role.
  • FE-02 Default task selectable per role.
  • FE-03 Archive tasks (hidden but retained for audit).
  • FE-04 Candidate uploads response as PDF or ZIP only.

3.10 Email Notifications & Templates

Templates (PTC editable): Screening shortlisted (internal), Fitment shortlisted/rejected, Technical shortlisted/rejected, PTC shortlisted/rejected, Founders shortlisted/rejected, Final Offer letter.

  • EM-01 WYSIWYG editor for templates; placeholders: {{candidate_name}}, {{role}}, {{interview_link}}, etc.
  • EM-02 Version history (last 10 versions) retained.

3.11 Views & Filters

  • VW-01 Two modes: Kanban (stage columns) and Tabular (sortable table).
  • VW-02 Filters: name, email, phone, date range added.
  • VW-03 Directors & Founders dashboard: candidate counts per stage, aging, rejection rates across requisitions.

3.12 Referral-Specific Requirements

  • REF-01 Referral form captures: employee ID/name, department, relationship, "Why do you recommend this candidate?" (free text).
  • REF-02 Referral template configurable by PTC.
  • REF-03 Referrer gets email when candidate is hired.

5. Data Entities (High-Level)

  • HiringRequisition – req_id, job_desc, scope, supporting_members[], status, approval_chain[]
  • Candidate – email (PK), name, phone, hashed_auth_code, created_at
  • Pipeline – pipeline_id, candidate_id, requisition_id, current_stage, status, referral_employee_id (opt)
  • StageHistory – stage_name, status, remarks, timestamp, actor_role, actor_id
  • FitmentTask – task_id, role_id, task_name, file_url, is_default, is_archived
  • EmailTemplate – template_id, name, subject, body_html, version

6. Assumptions & Constraints

  • Zoho People API provides real-time team/leadership hierarchy.
  • Zoho Bookings API supports virtual meeting links + calendar invites.
  • Email delivery infrastructure (transactional service) is available.
  • Initial Admin credentials securely hand-delivered.
  • File storage (CVs, task uploads) – S3-compatible, 50 MB limit per file.

7. Approval Workflow Diagram (Text)

Requisition approval: Draft → [Project Directors: any ✅] → Chief of Staff ✅ → [Founders: any ✅] → PTC ✅ → Active Hiring.
Rejection: any stage ❌ → Notify Creator → terminated.

Referral pipeline skip logic: Referral Created → PTC Override? Yes → Skip Fitment → Technical Interview; No → Fitment Evaluation → Technical Interview.
